### 課題（Pains）

**日々感じている課題**:

| # | 課題 | 深刻度 | 頻度 |
|---|------|-------|------|
| 1 | 案件情報が分散しておいしい案件を探しきれない（探索地獄） | 高 | 毎日 |
| 2 | 公告・仕様書を読んでも用語がわからず判断できない（読解地獄） | 高 | 案件ごと |
| 3 | 提出物の漏れ・形式不備・期限切れで無効になるリスクが怖い（手続き地獄） | 高 | 案件ごと |
| 4 | 入札価格の妥当性がわからない、相場が不明（見積地獄） | 中 | 案件ごと |
| 5 | 本業（弱電・IT）が忙しく入札に充てる時間が取れない | 高 | 毎日 |

**課題によって失っているもの**:
- 時間: 案件探し・仕様書読解だけで推定1日1〜2時間。本業を圧迫
- お金: ホットプラス月額会費。落札できないため投資回収ゼロ
- 機会: 統一資格を取得済みなのに1件も入札できていない（=全て機会損失）
- 精神的負担: 「何から手をつければいいかわからない」状態が続く不安

**課題を解決しようとした経験**:
| 試したこと | 結果 | なぜダメだったか |
|-----------|------|----------------|
| ホットプラス（入札コンサル）に入会 | ノウハウは得られるが、実行は全て自分 | 手順はわかっても手を動かす時間がない |
| データウェアネットで案件検索 | UIが使いにくく効率が悪い | 検索・絞り込みが煩雑、10,000件制限 |
| 調達ポータルの落札実績を確認 | 参考にはなるが相場感がつかめない | データが分散、分析ツールがない |

### ニーズ（Gains）

**求めていること**:

| # | ニーズ | 重要度 |
|---|-------|-------|
| 1 | 元請けとして自動で収益が出る入札の仕組み | 高 |
| 2 | 案件探索〜応札準備を半自動化し、自分の手間を最小化 | 高 |
| 3 | 下請けネットワーク（義父:軽運送、知人:内装・電気）を活用した実行体制 | 高 |
| 4 | 物品調達は通販仕入れの自動化で利益を出す | 中 |

**理想の状態**:
```
朝コーヒーを飲みながらダッシュボードを開くと、
昨夜自動収集された案件から「おすすめ3件」がカード表示されている。
AIが仕様書を読み、参加条件OK・スケジュール整理済み・チェックリスト作成済み。
「この案件いける」と思ったらワンクリックで応札準備が始まり、
義父の軽運送チームにも自動で概要が共有される。
月末には落札実績と失注分析がレポートで出てきて、
次月のスコアリングが自動で改善される。
本業のIT案件に集中しながら、入札事業が勝手に回っている。
```
