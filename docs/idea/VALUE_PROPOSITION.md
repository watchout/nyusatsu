**Functional Jobs（機能的な仕事）**:

| 3 | 提出物を漏れなく期限内に準備・提出する | 高 |
| 4 | 適正価格で入札し、落札する | 高 |

### 2.2 Pains（痛み）

> 仕事を達成する上での障害、課題、リスク

**痛みの深掘り**:

| # | 痛み | 深刻度 | 頻度 | 現状の対処 |
|---|------|-------|------|-----------|
| 1 | 案件情報が分散し、おいしい案件を探しきれない（探索地獄） | 高 | 毎日 | データウェアネットで手動検索。UIが使いにくく10,000件制限 |
| 2 | 公告・仕様書の用語がわからず、参加可否を判断できない（読解地獄） | 高 | 案件ごと | ホットプラスに都度相談。時間がかかる |
| 3 | 提出物の漏れ・形式不備・期限切れで無効になるリスク（手続き地獄） | 高 | 案件ごと | チェックリストなし。不安のまま進行 |
| 4 | 入札価格の妥当性がわからない。相場が不明（見積地獄） | 中 | 案件ごと | 落札実績を手動で調べるが、分析ツールがない |
| 5 | 負けた理由がわからず、次に活かせない（結果地獄） | 中 | 入札後 | 記録なし。感覚で次回を判断 |
| 6 | 本業が忙しく、入札業務に充てる時間が1日1〜2時間しかない | 高 | 毎日 | 後回しにして機会を逃す |

**痛みのカテゴリ**:

| カテゴリ | 具体的な痛み |
|---------|-------------|
| 時間の無駄 | 案件探索だけで推定1日1〜2時間。仕様書読解でさらに数時間 |
| お金の無駄 | ホットプラス月額会費を払いつつ、落札ゼロ＝投資回収ゼロ |
| 精神的ストレス | 「何から手をつければいいかわからない」状態が続く不安 |
| リスク・不安 | 提出漏れ・形式不備で無効入札になるかもしれない恐怖 |
| 機会損失 | 統一資格を取得済みなのに1件も入札できていない |

### 3.2 具体シーン：ペルソナA（金子）の毎朝の使い方

> 「入札業務が毎朝10分で回る」とはどういう状態か

```
■ 07:00 — コーヒーを淹れながらダッシュボードを開く（1分）

  昨夜の自動バッチ（06:00実行）で新着案件が収集済み。
  スコア80点以上の「おすすめ案件」が3件、カード形式で表示されている。

  カードには:
  - 案件名 / 発注機関 / 締切日
  - AIスコア（競争度30 + 規模25 + 期限25 + 適合度20 = 80点）
  - 参加可否: ✅ eligible（等級D・関東・物品販売 → 全条件OK）
  - リスク: ⚠️ 締切まで12日（余裕あり）

■ 07:02 — おすすめ3件を確認、1件を「応札予定」にする（3分）

  カードをタップして案件カード（AI読解結果）を確認:
  - 業務内容の要約（3行）
  - 参加条件チェック結果（5項目すべて✅）
  - 提出物リスト（7点）
  - 根拠: 仕様書 p.3「参加資格」、公告 §5「提出書類」（原文リンク付き）

  「応札予定」ボタンを押す → ステータスが planned に。
  → 自動でAIチェックリスト生成がキューに入る。

■ 07:05 — チェックリストを眺める（2分）

  チェックリストが自動生成済み:
  - □ 入札書（様式ダウンロード済み）
  - □ 委任状
  - □ 会社概要（前回提出分を再利用可）
  - □ 納入実績書（テンプレ自動生成済み）
  - □ ...
  - 📅 提出締切: 3/15（金）17:00
  - 📅 逆算スケジュール: 社内レビュー 3/12、最終確認 3/14

  「今日やること」はない。3/10から準備開始。

■ 07:07 — 価格分析を確認（2分）

  同じ発注機関の過去5件の落札実績:
  - 平均落札額: ¥480,000（予定価格の82%）
  - 参加社数: 3〜5社
  - 最低価格落札率: 78%

  → 入札価格の目安が見える。

■ 07:09 — 残り2件は「見送り」（1分）

  スコア65点の案件 → 締切5日で準備が間に合わない → 「見送り」+ 理由入力
  スコア70点の案件 → 納入先が九州 → 下請けNWがない → 「見送り」

■ 07:10 — ダッシュボードを閉じて本業へ

  合計10分。入札業務は「判断」だけ。探索・読解・準備はAIが済ませている。
```

### 3.3 時短の定義：どの工程が何分→何分になるか

> 「時短」を曖昧にしない。各工程の Before / After を明示する。

| # | 工程 | Before（手作業） | After（入札ラクダAI） | 削減率 |
|---|------|-----------------|---------------------|--------|
| 1 | **案件探索**（データウェアネット手動検索） | 60〜120分/日 | **0分**（自動収集。朝カード確認のみ） | 100% |
| 2 | **案件スクリーニング**（対象かどうか判断） | 30分/日（10件×3分） | **3分**（スコア順で上位3件を見るだけ） | 90% |
| 3 | **公告・仕様書の読解**（参加条件・提出物の理解） | 2〜4時間/件 | **2分/件**（AI読解カードを確認） | 98% |
| 4 | **参加可否の判断**（等級・品目・地域の照合） | 30分/件 | **0分**（自動判定。結果はカードに表示済み） | 100% |
| 5 | **提出物リスト作成**（仕様書から手動抽出） | 1時間/件 | **0分**（チェックリスト自動生成） | 100% |
| 6 | **スケジュール作成**（逆算して手動作成） | 30分/件 | **0分**（逆算スケジュール自動生成） | 100% |
| 7 | **価格調査**（落札実績を手動検索・集計） | 1〜2時間/件 | **2分/件**（価格分析を確認するだけ） | 95% |

**合計（1案件あたり）**: Before 5〜9時間 → After **7分** ※1

**毎日の案件チェック**: Before 1.5〜2時間 → After **10分**

> ※ 上記は「判断」と「確認」の時間。入札書の記入・押印・提出行為は含まない（Phase1対象外）。
>
> ※1 前提: 公告・仕様書取得済み（F-001完了）、AI読解完了（F-002完了）、company_profile登録済み、要確認案件は除く。つまり「日次運用の判断時間」としての7分であり、初回セットアップ時間は含まない。

### 3.4 不備ゼロの定義：何がゼロになるか

> 「不備ゼロ」を曖昧にしない。ゼロにする対象を列挙する。

| # | ゼロにする不備 | 現状（手作業）で起きうるミス | 入札ラクダAIでの防止策 |
|---|-------------|-------------------------|---------------------|
| 1 | **提出書類の漏れ** | 仕様書から必要書類を拾い切れない。後から気づいて慌てる | AI が仕様書・公告から提出物を自動抽出 → チェックリスト化。1つでも未チェックなら「未完了」表示 |
| 2 | **参加資格の見落とし** | 等級・地域・営業品目の不一致に気づかず応札 → 資格審査で落選 | 参加可否の自動判定（5条件: 統一資格・等級・品目・地域・免許）。不適格ならカードに ❌ 表示 |
| 3 | **提出期限の超過** | カレンダーに入れ忘れて期限切れ | 逆算スケジュール自動生成 + チェックリストに期限を明記。期限切れ案件は自動アーカイブ |
| 4 | **参加条件の誤読** | 仕様書の用語（「全省庁統一資格」「等級D以上」）を誤解して判断ミス | AI 読解結果に根拠箇所（ページ番号 + 原文リンク）を付与。誤読時に原文で即確認可能 |
| 5 | **落札データの未活用** | 過去の落札額を調べずに価格を決めて、高すぎ/安すぎで失注 | 類似案件の落札実績を自動表示。参加社数・落札率から相場感を提示 |

**「不備ゼロ」の定義**:
```
入札ラクダAI を使って応札準備を行った案件において、
以下の5カテゴリの不備発生件数がゼロであること:

1. 提出書類の漏れによる無効入札 = 0件
2. 参加資格の不適合による失格 = 0件
3. 提出期限超過による無効 = 0件
4. 参加条件の誤読による応札判断ミス = 0件
5. 価格データ未確認による大幅な価格乖離 = 0件

※ 検証方法: Phase1の全入札実績で計測。
   目標: 初回入札から6ヶ月間、上記5項目すべてゼロ。
```

### 3.5 価値→価格への翻訳：Starter / Pro / Biz

> §3.2〜3.4 で定義した「具体的な価値」を、Phase2 の3プランに翻訳する。

| 価値（§3.2〜3.4 で定義済み） | Starter | Pro（本命） | Biz（B向け） |
|---------------------------|---------|-----------|------------|
| **毎朝10分で案件チェック（§3.2）** | ✅ 案件カード＋スコアリング＋参加可否表示 | ✅ + AIスコア改善（失注学習反映） | ✅ + 複数担当者向け横断ダッシュボード |
| **案件探索 0分化（§3.3 #1）** | ✅ 自動収集（対象3領域） | ✅ + 対象領域拡張可能 | ✅ + 組織全体の案件プール共有 |
| **仕様書AI読解 2分化（§3.3 #3）** | ✅ AI読解カード＋根拠箇所リンク | ✅ + テンプレ自動生成（入札書・納入実績書等） | ✅ + 会員向けテンプレ（コンサル企業が配布可） |
| **チェックリスト自動生成（§3.3 #5）** | ✅ 案件ごとのチェックリスト＋逆算スケジュール | ✅ + 過去案件のチェックリスト再利用 | ✅ + 組織標準チェックリストの監査・承認フロー |
| **提出書類の漏れゼロ（§3.4 #1）** | ✅ AI抽出＋未完了アラート | ✅ + テンプレ事前入力で記入漏れも防止 | ✅ + 管理者によるダブルチェック機能 |
| **参加資格の見落としゼロ（§3.4 #2）** | ✅ 5条件自動判定 | ✅ + 外注先の資格情報も照合 | ✅ + 複数企業の資格を一元管理 |
| **価格分析で適正入札（§3.3 #7）** | ✅ 落札実績表示（過去5件） | ✅ + 失注分析→次回価格提案 | ✅ + 組織横断の価格戦略分析 |
| **失注→改善ループ** | ❌（記録のみ） | ✅ 失注理由分析＋スコアリング自動改善 | ✅ + 組織全体の勝率レポート |
| **外注/協力会社連携** | ❌ | ✅ 案件アサイン＋進捗共有 | ✅ + 権限管理＋監査ログ |

**プラン設計の原則**:
```
■ Starter = 「時短＋不備ゼロ」の基本価値（MVPの延長）

  提供する核心価値:
  - 案件探索の工数ゼロ化（§3.3 #1: 60〜120分 → 0分）
  - 仕様書読解の工数98%削減（§3.3 #3: 2〜4時間 → 2分）
  - 提出漏れ・資格見落としゼロ（§3.4 #1, #2）

  → 「入札を始めたい」人が、最低限これだけで入札事業を回せる

■ Pro = 「勝率向上＋業務拡大」の成長価値（本命プラン）

  Starter の全価値に加えて:
  - 失注→学習→スコアリング改善ループ（使うほど賢くなる）
  - テンプレ自動生成で「準備」の工数もゼロに近づける
  - 外注/協力会社への案件アサインで事業規模を拡大

  → 「月2件の落札」を「月5件」に増やしたい人の成長エンジン

■ Biz = 「組織運用＋品質統制」のスケール価値（B向け）

  Pro の全価値に加えて:
  - 会員向けテンプレ配布（コンサル企業がクライアントに提供）
  - 横断ダッシュボード（複数担当者の案件状況を一画面で）
  - 監査/権限管理（誰がいつ何を承認したかの証跡）

  → ホットプラスのようなコンサル企業がツールとして導入
```

**想定価格レンジ（Phase1 の計測後に確定）**:

| プラン | 提供価値の金額換算 | 想定月額 | 根拠 |
|-------|-----------------|---------|------|
| Starter | 時短価値: 月40時間×¥2,000/h = ¥80,000相当 | ¥9,800〜14,800 | 時短価値の12〜18%。入札王と同価格帯だが価値が段違い |
| Pro | 時短＋勝率向上: 月1件追加落札 = ¥300,000〜相当 | ¥29,800〜49,800 | 追加落札額の10〜16%。コンサル月額と同等で自動化 |
| Biz | 組織効率化: 担当者3名×月40時間 = ¥240,000相当 | ¥98,000〜 | 人件費削減の40%。ホットプラスのビジネスモデルを代替 |

> ※ 価格はPhase1の自社運用で「実際の時短時間」「落札率」「不備発生率」を計測してから確定する。
> 上記は計測前の仮説であり、Phase1 の成果指標で検証する。

### 3.6 Pain Relievers（痛みの解消）

> 顧客の痛みをどう解消するか

| 痛み | 解消方法 | 解消度 |
|------|---------|-------|
| 探索地獄（案件が分散・UIが使いにくい） | 案件自動収集＋AIフィルタリング。対象3領域に絞り込んだ自動巡回で毎朝スコア付きカード表示 | 完全 |
| 読解地獄（用語・条件がわからない） | 公告・仕様書のAI読解。要点抽出＋用語解説＋根拠箇所リンク付き | 部分的（AI精度に依存。人間の最終確認を前提） |
| 手続き地獄（提出漏れ・形式不備・期限） | チェックリスト＋テンプレート自動生成＋期限リマインド | 完全 |
| 見積地獄（相場がわからない） | 落札実績データの価格分析。過去の落札金額・参加社数で相場感を提供 | 部分的（データ蓄積量に依存） |
| 結果地獄（負けた理由が残らない） | 失注情報の自動記録＋分析（Phase2で改善提案まで拡張） | 部分的（Phase1は記録まで） |
| 本業が忙しく時間がない | 全工程の半自動化で、案件チェックを1日10分以内に短縮 | 完全 |

## 4. フィット分析

### 4.1 Problem-Solution Fit

> 課題と解決策が合っているか

| 痛み/利得 | 対応する機能 | フィット度 | コメント |
|----------|-------------|-----------|---------|
| 探索地獄 → 毎朝10分 | 案件自動収集＋AIフィルタ＋スコアリング | ◎ | 手動巡回を完全に置き換え |
| 読解地獄 → 要点自動整理 | 公告・仕様書AI読解＋案件カード化 | ○ | AI精度に依存。根拠保持で信頼性を担保 |
| 手続き地獄 → チェックリスト | チェックリスト＋テンプレ自動生成 | ◎ | ホットプラスのノウハウをルール化 |
| 見積地獄 → 相場感 | 落札実績データ＋価格分析 | ○ | データ蓄積量に依存。Phase1で精度検証 |
| 結果地獄 → 学習ループ | 失注記録＋分析（Phase2で拡張） | △ | Phase1は記録まで。改善提案はPhase2 |
| 時間不足 → 半自動化 | 全5機能の統合 | ◎ | 1日1〜2時間 → 10分。最大の価値 |

**凡例**:
- ◎: 完全にフィット
- ○: 概ねフィット
- △: 部分的にフィット
- ×: フィットしない

### 4.2 フィットスコア

| 指標 | スコア | 判定 |
|------|-------|------|
| 重要な痛みへの対応率 | 5/6（結果地獄はPhase2で本格対応） | 良好 |
| 重要な利得への対応率 | 4/5（月2件落札はPhase1で検証） | 良好 |
| 総合フィット度 | 9/10 | **Go** |

> **判定根拠**: 6つの痛みのうち5つにMVPで対応。残り1つ（結果地獄）もPhase2で拡張予定。
> Phase1の最大リスクは「AI読解の精度」と「データ蓄積量」だが、根拠保持＋人間確認の設計で軽減。

## 7. 競合との比較

### 7.1 価値提案の比較

| 価値提案要素 | 入札ラクダAI | NJSS | 入札王 | ホットプラス |
|-------------|-------------|------|-------|------------|

| 主要な利得の創出 | 時間短縮＋チェックリスト＋価格分析＋学習 | 案件発見効率の向上 | 新着通知の効率化 | ノウハウ提供＋個別サポート |
| 独自の価値 | ✅ 実務の半自動化（探した後） | ❌ | ❌ | △（人手で対応。ツール化されていない） |

### 7.2 差別化ポイント

```
競合にはなく、入札ラクダAIだけが提供できる価値:

1. 仕様書のAI読解＋参加可否自動判定
   → 「読めない」「判断できない」を直接解消する。競合は検索まで

2. 案件ごとのチェックリスト＋テンプレート自動生成
   → 「何を出せばいいかわからない」を消す。競合はテンプレ提供なし

3. 落札/失注データの蓄積による勝ちパターン学習
   → 使うほどスコアリング精度が上がる。競合にはデータ蓄積の仕組みがない
```
