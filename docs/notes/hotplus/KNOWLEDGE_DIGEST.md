# ホットプラス ナレッジダイジェスト — 入札の流れ（全体構造）

> 出典: ホットプラス会員サイト「入札の流れ」（全17ページ）
> 作成日: 2026-02-16

---

## 入札フロー全体像

```
STEP1 案件検索    → データウェアネットでキーワード検索・絞り込み
       ↓
STEP2 公告確認    → 参加条件・スケジュール・仕様書取り寄せ
       ↓
STEP3 仕様書確認  → 業務内容・提出物の確認
       ↓
  [同等品申請]    → 物品案件の場合、安価な同等品で原価を下げる（入札書提出前）
       ↓
STEP4 入札参加    → 入札書記入・封筒作成・提出
       ↓
STEP5 結果確認    → 落札/失注の確認・失注分析
       ↓
STEP6 業務履行    → 物品配送 or 役務実行 → 請求書送付
```

## 自動化の観点から抽出したキーポイント

### 🔍 STEP1: 案件検索（自動化の最優先対象）
- **データソース**: データウェアネット（入札検索タブ）
- **検索方法**: キーワード＋スペース区切り、「いずれかを含む」に変更が必須
- **絞り込み**: 等級（A〜D）、入札受付中、都道府県、発注機関
- **制約**: 10,000件以上はヒットしない → 絞り込み必須
- **出力**: CSV出力可能（地方自治体の案件を省くことも可能）
- → **自動化ポイント**: キーワード検索・絞り込み・CSV取得を自動巡回で実現可能

### 📋 STEP2: 公告確認（判定ロジックの核）
- **参加条件チェック（4項目）**:
  1. 「全省庁統一資格」の記載があるか
  2. 営業品目（ほとんどは「その他」でOK）
  3. 等級（ほとんどは「D等級」でOK）
  4. 競争参加地域
- **その他の参加資格**: 許認可・実績・ISO/Pマーク・所在地・物品の製造
- **スケジュール項目**: 仕様説明会日時、入札書提出期限、開札日時、同等品申請期限、履行期限
- → **自動化ポイント**: 公告テキストから参加可否を自動判定可能（ルールベース）

### 📖 STEP3: 仕様書確認（AI読解の対象）
- **物品の販売**: 納期、納入場所、下見積もりの有無、物品一覧
- **役務の提供**: スポット/年間契約、作業内容、説明会の有無
- **提出物**: 入札参加時の書類 + 作業履行時の書類（2種類）
- **重要**: 下見積もり書には入札参加額より高い金額を記入して提出すること
- → **自動化ポイント**: 仕様書PDFの要約・チェックリスト生成

### 🔄 同等品申請（物品案件の勝率を大幅に上げる）
- **タイミング**: 入札書提出前
- **手順**: 申請可否確認 → 要求品の価格調査 → スペック比較 → 安価な同等品を探す → 申請書作成・提出
- **探し方**: 同一メーカーの旧型品/後継品、類似品表示の活用、安価メーカーでの検索、仕入れ業者への依頼
- **重要**: いくつでも申請可能。少しでも承認される可能性があるものは申請すべき
- → **自動化ポイント**: 要求品のスペック抽出→通販サイトで同等品自動検索→比較表生成

### ✍️ STEP4: 入札参加（テンプレ化可能）
- **入札書**: 発注機関のひな形を使用（機関が定める形式以外は無効になる場合あり）
- **封筒**: 内封筒（長3）+ 外封筒の二重構造
- **提出方法**: 書留 or レターパック（事前許可要）
- → **自動化ポイント**: 入札書・封筒の記入テンプレート自動生成

### 📊 STEP5: 結果確認（学習ループの入口）
- **確認方法**: 郵送→電話連絡待ち / 現地→その場で / 電子→メール＋調達ポータル
- **失注時に確認すべき3点**: 落札金額、落札業者、参加社数
- **失注分析**: 公告・仕様書・自身の入札金額・落札業者のHP/URLをまとめて記録
- → **自動化ポイント**: 失注情報の自動記録＋分析＋次回改善提案

### 💰 STEP6: 業務履行・請求（元請けオペレーション）
- **物品の販売**:
  - 通販サイトから直接配送可能（ギフト設定で明細書を同封しない）
  - 担当者に事前にお届け日・配送元を連絡すること
  - 設置・納入検査・配送業者指定の有無を確認
- **役務の提供**:
  - 作業者名簿の提出が必要
  - 下請け業者は「落札業者の人間として」記入・入場
- **請求**:
  - こちらから送らない限り振込はされない
  - 請求書受領後30日以内に振込
  - 雛形は官公庁からもらえる（なければネットのテンプレで可）
- → **自動化ポイント**: 配送手配・作業者名簿・請求書の自動生成

## 入札ラクダAI への設計示唆

### MVP（レベル2）で実装すべき機能マッピング

| フロー段階 | 自動化対象 | 優先度 |
|-----------|-----------|--------|
| STEP1 | キーワード検索の自動巡回・案件カード化 | P0 |
| STEP2 | 参加条件の自動判定（全省庁統一資格チェック） | P0 |
| STEP2 | スケジュールの自動抽出・期限管理 | P0 |
| STEP3 | 仕様書の要約・チェックリスト生成 | P0 |
| 同等品 | 要求品の価格・スペック自動調査 | P1 |
| 同等品 | 同等品候補の自動検索・比較表生成 | P1 |
| STEP4 | 入札書・封筒テンプレートの自動生成 | P1 |
| STEP5 | 失注情報の記録・分析・改善提案 | P1 |
| STEP6 | 請求書テンプレート自動生成 | P2 |
| STEP6 | 配送手配・作業者名簿の自動生成 | P2 |

### 判定ロジック設計のヒント

**参加可否の自動判定ルール:**
```
IF 公告に「全省庁統一資格」記載あり
  AND 等級が自身の等級以上（D等級はほぼ全て参加可能）
  AND 営業品目が「その他」で参加可能 or 指定品目を保有
  AND 競争参加地域に該当
  AND 許認可要件を満たす（or 要件なし）
THEN → 参加可能
ELSE → 参加不可（不可理由を表示）
```

**案件スコアリングの観点:**
- 履行期限の余裕度
- 同等品申請の可否（物品案件の場合、利益率向上の可能性）
- 下請け業者の必要性と手配難易度
- 過去の落札傾向（参加社数・落札金額）
